# 🛒 Desafio SQL: Modelagem e Consultas para E-commerce
> **Bootcamp Heineken/DIO:** Inteligência Artificial Aplicada a Dados com Copilot.

Este repositório contém a resolução do desafio prático de banco de dados, focado na construção de um ecossistema de E-commerce. O projeto abrange desde a concepção do modelo lógico até a execução de queries complexas para extração de insights de negócio.

---

## 🎯 Objetivo do Projeto
Demonstrar o domínio em bancos de dados relacionais, estruturando um esquema capaz de gerenciar:
- Clientes (Pessoa Física e Jurídica).
- Processamento de pedidos e controle de estoque.
- Gestão de fornecedores e logística de entrega.
- Métodos de pagamento diversificados.

---

## 📂 Estrutura do Repositório

- **`CRIACAO_DAS_TABELAS`**: Script SQL (DDL) com a estrutura das tabelas, chaves primárias e estrangeiras.
- **`INCLUSAO_DE_DADOS`**: Script SQL (DML) com dados fictícios para testes de consistência.
- **`MODELO_LOGICO.jpg`**: Diagrama Entidade-Relacionamento (DER) detalhando as conexões entre as entidades.
- **`SCRIPTS_DE_CONSULTAS`**: Conjunto de queries SQL focadas em responder perguntas de negócio.
- **`QUERY1.png` a `QUERY9.png`**: Screenshots dos resultados das consultas executadas no banco de dados.

---

## 🧠 Modelagem Lógica
O modelo foi estruturado para suportar a complexidade de um e-commerce moderno, incluindo a distinção entre tipos de clientes e a relação entre produtos, fornecedores e estoques.

*(Para visualizar o diagrama completo, acesse o arquivo `MODELO_LOGICO.jpg` neste repositório)*

---

## 📊 Queries e Insights de Negócio
As consultas desenvolvidas visam responder KPIs essenciais, como:
- **Recuperação de dados:** Filtros específicos com `WHERE`.
- **Campos calculados:** Geração de métricas durante a consulta.
- **Ordenação:** Organização de dados com `ORDER BY`.
- **Filtros de Grupo:** Uso de `HAVING` para análise de dados agregados.
- **Junções:** Integração de múltiplas tabelas via `JOIN` para uma visão 360º do pedido.
